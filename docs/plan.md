# StudyBuddyAI — 设计方案

> 基于 Harness Engineering 思维设计的 AI 作业监督系统

## 一、核心洞察

家长的需求本质上是一个**长时间自主运行的 AI Agent**，对应 Harness Engineering 的 Ralph 循环模式：

- **裸模型（豆包C端）** = 有智能，但没有状态、流程编排、反馈回路
- **Agent = Model + Harness** = 模型 + 任务状态机 + 背压门控 + 反馈回路

需要的不是更强的模型，而是一个 **Harness（驭缰系统）** 来编排模型的能力。

## 二、系统架构

```
家长端（微信/Web）
    ↓ 1.发送作业清单
后端服务
  ├── Orchestrator 任务编排器
  ├── 状态机管理器
  ├── 会话状态存储
  └── 报告生成器
    ↓ 2.生成计划 → 3.驱动状态流转 → 4.下发阶段指令
AI API 层
  ├── Gemini Live API（语音+视觉，首选）
  ├── 豆包实时语音大模型（备选）
  └── 豆包视觉理解（备选）
    ↕ 实时音视频流
孩子端（手机/小智机器人）
  ├── 摄像头
  ├── 麦克风
  └── 扬声器
    ↓ 5.任务全部完成
报告推送 → 家长端
```

## 三、核心状态机（7个状态）

| 状态 | 含义 | 触发转换 |
|------|------|---------|
| PlanReady | 家长已提交作业，AI 已生成计划 | 孩子端连接 → TaskBriefing |
| TaskBriefing | AI 语音告知当前任务内容和时间 | 30秒后 → Studying |
| Studying | 监督中：视觉检测+计时+响应提问 | 时间到/AI判断完成 → BreakTime |
| Nudge | 检测到异常，主动语音提醒 | 孩子回归 → Studying |
| BreakTime | 休息：倒计时，到点提醒下一任务 | 休息结束 → TaskBriefing / AllDone |
| AllDone | 全部完成：语音表扬，回顾表现 | 自动 → ReportSent |
| ReportSent | 生成报告推送家长 | 终态 |

**关键：状态流转由 Orchestrator 驱动，不依赖孩子操作。**

## 四、技术选型

### AI API

| 能力 | 首选 | 备选 | 理由 |
|------|------|------|------|
| 语音+视觉实时交互 | Gemini Live API | - | 唯一原生支持视频流+语音+文本三模态实时交互 |
| 语音对话 | Gemini Live | 豆包实时语音 | 豆包中文语音质量更高(4.36/5) |
| 视觉检测 | Gemini Live | 豆包视觉理解 | 豆包视觉每千token仅0.003元 |

### 组合策略

- **方案A（首选）**：Gemini Live API 一体化 — 一个API搞定语音+视觉+对话
- **方案B（备选）**：豆包语音 + 豆包视觉 分离架构 — 中文效果最佳
- 通过抽象层隔离，切换成本很低

### 技术栈

- 后端：Python (FastAPI)
- 状态存储：SQLite + JSON 文件
- 通信协议：WebSocket
- 客户端（初期）：手机浏览器 PWA
- 客户端（后期）：小智 ESP32 机器人
- 家长端：Web 页面（后续可接微信）

## 五、Gemini Live API 关键限制与应对

| 限制 | 应对策略 |
|------|---------|
| 视频会话2分钟上限 | ContextWindowCompressionConfig 滑动窗口 + SessionResumptionConfig 跨连接恢复 |
| WebSocket连接10分钟上限 | Orchestrator 自动管理 session 续期，对孩子透明 |
| 视频帧率 ≤1FPS | 足够用于坐姿/注意力检测，不需要高帧率 |

## 六、Harness Engineering 六大原则映射

1. **仓库即记录系统** — 会话状态、行为日志、决策记录全部持久化到 data/
2. **地图而非手册** — AGENTS.md 入口 ≤100行，Prompt 按角色拆分
3. **机械化执行** — 状态机本身就是约束，AI 只能在合法动作范围内行动
4. **智能体可读性** — 统一 CLI 入口 + --json 输出
5. **反馈回路** — 视觉反馈（坐姿评分）、时间反馈（定时器）、家长反馈（报告评价）
6. **熵管理** — Prompt 模板版本化，配置与 Prompt 分离

## 七、MVP 范围

**做：**
1. 家长通过 Web 页面输入作业（文本格式）
2. AI 生成时间计划（含每科时间+休息）
3. 手机浏览器 Web App，开启摄像头+麦克风
4. AI 自动推进：任务介绍 → 监督写作业 → 休息 → 下一任务
5. 监督期间检测注意力/坐姿，主动语音提醒
6. 完成后生成文字报告，家长 Web 页面查看

**不做（后续迭代）：**
- 微信推送
- 小智机器人适配
- 多孩子支持
- 历史数据分析
- 作业图片识别

## 八、设备演进路径

1. **Phase 1**：手机架在支架上（PWA Web App）
2. **Phase 2**：小智 ESP32 机器人（通过 WebSocket/MQTT 接入后端）
3. **Phase 3**：专用学习桌设备（摄像头+屏幕一体化）
